# gost-stunnel-CSP

ГОСТ туннель для подключения к госсервисам:
- ГИС ЖКХ
- Росфинмониторинг
- и т.д.

Контейнер терминирует/инициирует ГОСТ-TLS сессию через КриптоПро CSP и
проксирует трафик к удалённому хосту (при необходимости — через HTTP-прокси).
Клиент подключается к локальному порту `8080`.


## Структура репозитория
```
.
├── .docker
│   ├── Dockerfile
│   ├── stunnel.conf
│   └── scripts
│       ├── entrypoint.sh
│       └── stunnel-socat.sh
└── resource                                    # секреты, в образ не коммитятся
    ├── root.cer                                # корневой/клиентский сертификат
    └── root.zip                                # архив: сертификат + ключевой контейнер
```

> Пакеты КриптоПро CSP больше не вендорятся в репозиторий — они ставятся из
> закрытого apt-репозитория КриптоПро на этапе сборки (см. «Сборку»).

### Папка resource
Содержит секретный материал клиента. В систему контроля версий не попадает
(см. `resource/.gitignore`); сертификаты и ключ запекаются в образ на этапе
сборки (design B).

Содержимое:
- **root.cer** — корневой сертификат УЦ.
- **root.zip** — архив с клиентским сертификатом и ключевым контейнером.

#### Подготовка архива root.zip

1. Установите КриптоПро CSP и импортируйте ключевой контейнер с носителя
   (флешка / реестр) на машину, где готовите ресурсы.
2. Экспортируйте сертификат клиента из контейнера.
3. После экспорта сертификата из контейнера подготовьте архив:
   - `root.cer` — сертификат из контейнера.
   - `name.000` — ключевой контейнер (папка).

**Важно!** Если в имени контейнера используется кириллица, нужно изменить его на
латиницу. Откройте файл `name.key`, измените имя и сохраните файл.

Создайте архив `root.zip` и скопируйте его в папку `resource`.


## Сборка

Образ собирается через **BuildKit**. Все чувствительные данные передаются как
**секреты** (не попадают в историю слоёв образа): учётные данные cryptopro.ru для
apt-репозитория, лицензия КриптоПро и PIN ключевого контейнера. Пакеты КриптоПро CSP
ставятся из закрытого apt-репозитория КриптоПро (поддерживает amd64 и arm64,
архитектура определяется автоматически — `--build-arg TARGETARCH` не нужен).

### 1. Прокиньте переменные окружения

Перед сборкой задайте переменные окружения (из них BuildKit берёт значения секретов):

```bash
# Учётные данные аккаунта cryptopro.ru для apt-репозитория, формат USERNAME:PASSWORD.
# ОБЯЗАТЕЛЬНО — без них пакеты КриптоПро не скачать.
export CRYPTOPRO_CREDS='user@example.com:ВашПароль'

# Лицензия КриптоПро (опционально; пусто -> trial/Demo-режим).
export CRYPTOPRO_LICENSE='XXXXX-XXXXX-XXXXX-XXXXX-XXXXX'

# PIN ключевого контейнера (опционально; нужен, если контейнер защищён PIN).
export CERTIFICATE_PIN='<PIN контейнера>'
```

> Пароль может содержать спецсимволы (`@ ? % )` и т.п.) — оборачивайте значение в
> одинарные кавычки.

### 2. Соберите образ

```bash
DOCKER_BUILDKIT=1 docker build \
  --secret id=cryptopro_creds,env=CRYPTOPRO_CREDS \
  --secret id=cryptopro_license,env=CRYPTOPRO_LICENSE \
  --secret id=certificate_pin,env=CERTIFICATE_PIN \
  -f .docker/Dockerfile -t rfm-tunel .
```

| Секрет | env-переменная | Обязателен | Назначение |
|--------|----------------|------------|------------|
| `cryptopro_creds` | `CRYPTOPRO_CREDS` | да | доступ к apt-репозиторию КриптоПро (`USERNAME:PASSWORD`) |
| `cryptopro_license` | `CRYPTOPRO_LICENSE` | нет | лицензия КриптоПро (пусто → trial) |
| `certificate_pin` | `CERTIFICATE_PIN` | нет | PIN ключевого контейнера |

> Лицензия устанавливается отдельным финальным слоем образа, поэтому её замена
> не инвалидирует тяжёлые слои (установка пакетов, импорт сертификатов).


## Запуск

### Общий синтаксис
```bash
docker run -p 8080:8080 \
  -e STUNNEL_HOST=<TARGET-HOST>:443 \
  -e STUNNEL_HTTP_PROXY=<HTTP-PROXY-HOST> \
  -e STUNNEL_HTTP_PROXY_PORT=<PORT> \
  -e STUNNEL_HTTP_PROXY_CREDENTIALS=<LOGIN:PASS> \
  rfm-tunel
```

Переменные окружения:
- **STUNNEL_HOST** — целевой ГОСТ-TLS хост `host:port` (обязательная).
- **STUNNEL_HTTP_PROXY**, **STUNNEL_HTTP_PROXY_PORT**, **STUNNEL_HTTP_PROXY_CREDENTIALS**
  — параметры HTTP-прокси (опционально).
- **STUNNEL_DEBUG_LEVEL** — уровень логирования stunnel (опционально).

### Росфинмониторинг
```bash
docker run -p 8080:8080 -e STUNNEL_HOST=portal.fedsfm.ru:8081 rfm-tunel
```

**Проверка:**
```bash
curl http://localhost:8080/Services/fedsfm-service
```
